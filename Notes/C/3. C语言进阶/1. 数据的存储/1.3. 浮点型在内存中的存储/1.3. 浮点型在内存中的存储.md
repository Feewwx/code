```c
#include <stdio.h>

int main() {
    int n = 9;
    float *pFloat = (float*)&n;
    printf("n的值为:%d\n", n);             // n的值为:9
    printf("pFloat的值为:%f\n", *pFloat);  // pFloat的值为:0.000000

    printf("n的值为:%d\n", n);             // n的值为:1091567616
    printf("*pFloat的值为:%f\n", *pFloat); // *pFloat的值为:9.000000
    return 0;
}
```

**浮点数储存规则:**

根据国际标准IEEE (电气和电子工程协会) 754,任意一个二进制浮点数V可以表示成下面的形式:

`(-1)^S * 1.M * 2^E`
`(-1)^S` 表示符号位,当S=0,V为整数;当S=1,V为负数
`1.M` 表示有效数字,大于等于1,小于2
`2^E` 表示指数位

**例:**
- V = 5.0f => 101.0 => 1.01 * 2^2
- V = (-1)^0 * 1.01 * 2^2
- S = 0 M = 01 E = 2
####
- V = 9.5f => 1001.101 => 1001.1 => 1.0011 * 2^3
- V = (-1)^0 * 1.0011 * 2^3
- S = 0 M = 0011 E = 3
```
2^3|  2^1|  2^-1|   2^-3|
   1  0  0  1 . 1   0   1
   2^2|  2^0|   2^-2|
```
> 因为 2^-1 = 0.5 所以后面的01就不能再要了

V = 9.6f => 1001.10..............
> 无法精确存储,float 中实际存储的值大约是 9.600000381469727

**`float` (32位)在内存中的存储方式:**

|-S(1bit)-|-E(8bit)-|-M(23bit)-|

**`double`(64位)在内存中的存储方式:**

|-S(1bit)-|-E(11bit)-|-M(52bit)-|

**两个特别规定:**

- 因为M一般为1.XXXXXXX,其中XXXXXXXX表示小数部分
所以,存储的时候,开头的1可以省去,这样就可以多存储一位小数
####
- 因为E有时候为负数,所以存入内存时,必须加上一个中间数
比如float中E为8位,它的取值范围为0-255,所以中间数为127
比如double中E为11位,它的取值范围为0-2047,所以中间数为1023

**例:**
- V = 0.5f => 0.1
- V = (-1)^0 * 1.0 * 2^(-1)
- S = 0 M = 0 E = -1 + 127 = 126 / + 1023 = 1022
####
- V = 5.5f => 101.1
- V = (-1)^0 * 1.011 * 2^(2)
`0 10000001 01100000000000000000000`
浮点数在内存中就是这样存储的

**指数E从内存中取出还能分为三种情况:**

- E不全为0或不全为1
正常情况,E的计算值-127(1023)拿到真实值,在将有效数字M前加上第一位的1
####
- E为全0
因为这是一个在坐标轴上很接近0的数字,所以M前补0
####
- E为全1
表示正负无穷大

**现在解释前面的代码**
```c
int n = 9;
```
9的补码为`00000000000000000000000000001001`
```c
float *pFloat = (float*)&n;
```
现在他被强制类型转换为float,所以计算机会用指针的方式来看他
`0 00000000 00000000000000000001001`
E为全0,所以这是一个很小的数字,因此输出0.000000
```c
*pFloat = 9.0;
```
现在以字符的形式存入9.0
`0 10000010 00100000000000000000000`
```c
printf("n的值为:%d\n", n);
```
又强制以int类型输出了,计算机以整形的方式看他
`01000001000100000000000000000000`转换为十进制为1091567616
因此输出1091567616
